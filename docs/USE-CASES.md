## 実装パターンリファレンス

### 典型的なccmux auto 呼び出しパターン

```bash
# パターン1: GitHub issue → 自動セッション（n8n経由）
# GitHub issue作成 → n8n → ccmux serve → ccmux auto <issue-N>

# パターン2: ループモード（ralph-loop）
ccmux auto \
  --loop --max-iter 50 \
  --until CCMUX_COMPLETE \
  --prompt-file task.md \
  my-session

# パターン3: 一晩実行（overnight）
ccmux auto \
  --loop --max-iter 200 \
  --sandbox \                    # bubblewrapサンドボックス（Linux）
  --prompt-file overnight.md \
  overnight-$(date +%Y%m%d)

# パターン4: 並列スプリント（3ワークツリー同時）
for feature in auth search ui; do
  ccmux auto \
    --prompt "Implement $feature feature per spec/$feature.md. CCMUX_COMPLETE when done." \
    sprint-$feature &
done
wait

# パターン5: 完了後にreflect → CLAUDE.md自己改善
ccmux close my-session
ccmux reflect my-session --apply
```
